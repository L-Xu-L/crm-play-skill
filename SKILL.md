# CRM 客户画像搜索与批量导入

通过自然语言描述客户画像，搜索匹配的潜在客户，确认后批量导入 CRM 系统。

## 核心流程

```
1. 用户描述画像 → 2. 搜索客户 → 3. 展示列表 → 4. 用户确认 → 5. 批量导入
```

## 可用工具

### search_customer_profile

根据画像描述搜索匹配的客户。

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| profile_description | string | 是 | 画像描述，如「日本地区、年采购额>100万」 |
| limit | int | 否 | 返回条数，默认 10 |

**返回：** 客户列表（名称、联系人、邮箱、电话、地址、相似度）

**示例：**
```
Input: 帮我找日本地区的酒店客户
调用: search_customer_profile(profile_description="日本地区的酒店客户", limit=10)
```

### batch_import_customer

将客户批量导入 CRM 作为潜在客户。

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customers | array | 是 | 客户列表 |

每个客户包含：
- `name`: 客户/公司名称（必填）
- `contact_name`: 联系人姓名（必填）
- `contact_email`: 联系人邮箱（必填）
- `contact_phone`: 联系人电话（必填）
- `contact_phone_prefix`: 电话区号（默认 +86）
- `country_code`: 国家编码（默认 CN）
- `address`: 地址
- `remark`: 备注

**返回：** 成功数、失败数、失败明细

## 使用指南

### 步骤 1：理解画像需求

用户可能用这些方式描述画像：
- 地区：「日本」「东南亚」「欧洲」
- 业务：「酒店」「旅游」「商务」
- 规模：「年采购额>100万」「大型企业」

### 步骤 2：搜索并展示

调用 `search_customer_profile` 后，用以下格式展示：

```
根据您的画像「{画像描述}」，找到以下 {数量} 位匹配客户：

1. **{公司名称}** (相似度: {similarity}%)
   - 联系人：{contact_name}
   - 邮箱：{contact_email}
   - 电话：{contact_phone_prefix} {contact_phone}
   - 地址：{address}

2. ...

是否要将这些客户录入 CRM 系统？
```

### 步骤 3：确认导入

当用户回复肯定词（「是」「好的」「录入」「导入」「确认」），调用 `batch_import_customer`。

### 步骤 4：反馈结果

```
导入完成！成功 {success} 条，失败 {fail} 条。
{如有失败，列出失败明细}
```

## 注意事项

1. **邮箱去重**：系统按联系邮箱去重，已存在的邮箱会导入失败
2. **必填字段**：客户名称、联系人、联系邮箱、联系电话为必填
3. **默认值**：国家编码默认 CN，电话区号默认 +86
4. **客户类型**：导入的客户默认为「潜在客户」，跟进状态为「初步接触」
